# 奇美醫院 每日電子報自動化流程

## 架構概覽

```
每天 08:30 CST
      │
      ▼
GitHub Actions (cron: '30 0 * * *')
      │
      ├─ 1. fetch RSS 新聞（衛福部、聯合報、康健）
      │
      ├─ 2. 呼叫 Claude API（Anthropic）
      │        System Prompt = chimei-epaper skill
      │        → 回傳 JSON（深耕專題 / 專家講堂 / 活動資訊）
      │
      ├─ 3. Jinja2 套用 HTML 模板
      │        → output/index.html
      │
      └─ 4. 推送到 GitHub Pages（gh-pages 分支）
               → https://yourorg.github.io/daily-brief/
```

## 設定步驟

### 1. Fork / Clone 這個 repo

### 2. 在 GitHub Repo → Settings → Secrets 加入：

| Secret 名稱         | 說明                          |
|---------------------|-------------------------------|
| `ANTHROPIC_API_KEY` | 你的 Anthropic API Key        |

### 3. 開啟 GitHub Pages
- Settings → Pages → Source → **gh-pages 分支**

### 4. 等待 08:30（台灣時間）自動觸發
- 或到 Actions → 手動點「Run workflow」測試

## 目錄結構

```
.
├── .github/
│   └── workflows/
│       └── daily-brief.yml     ← GitHub Actions 排程設定
├── scripts/
│   └── generate_brief.py       ← 主程式
├── templates/
│   └── brief.html              ← HTML 模板（Jinja2）
├── output/                     ← 自動產生（勿手動編輯）
│   ├── index.html
│   └── YYYY-MM-DD.json
└── requirements.txt
```

## 客製化

### 換新聞來源
在 `scripts/generate_brief.py` 的 `RSS_FEEDS` 清單新增 RSS URL。

### 調整 Claude 提示
修改 `scripts/generate_brief.py` 裡的 `SYSTEM_PROMPT` 字串。

### 推送到自架網站（非 GitHub Pages）
將 `daily-brief.yml` 裡的 **Deploy to GitHub Pages** 步驟換成：
```yaml
- name: Deploy via SFTP
  uses: wlixcc/SFTP-Deploy-Action@v1.2.4
  with:
    username: ${{ secrets.FTP_USER }}
    server: ${{ secrets.FTP_HOST }}
    password: ${{ secrets.FTP_PASS }}
    local_path: ./output/*
    remote_path: /var/www/html/daily-brief/
```
